🧠

Come funziona un LLM

Pre-training su miliardi di testi, poi fine-tuning e RLHF. Genera testo token per token basandosi su probabilità statistiche.

🔤

Token & Context Window

Un token corrisponde generalmente a 3/4 di parola, ma dipende dal modello usato. Il modello "ricorda" solo ciò che rientra nella sua finestra di contesto — dai 4K ai 200K token.

🌡️

Temperature

Temperature 0 = risposta deterministica. Valori alti = più creatività e variabilità nell'output generato.

⚡

Prompt = Input AI

La qualità dell'output dipende direttamente dal prompt. Una richiesta vaga produce una risposta vaga — prima lezione.

⚠️

Allucinazioni

I modelli generano fatti plausibili ma falsi. Non sono calcolatori: producono testo statisticamente coerente, non "verità".

Agenti AI

Compie azioni pratiche delineate, superando i limiti strutturali dell'intelligenza artificiale generica.

Sezione 03

Prompt Engineering

Le tecniche che abbiamo imparato e poi applicato al progetto CV.

Zero-Shot

Nessun esempio nel prompt: il modello usa solo le istruzioni fornite. Funziona per task noti, ma produce risultati meno prevedibili sul formato.

Few-Shot

2–5 esempi di input/output guidano il modello verso il formato desiderato. La tecnica più potente per output strutturati come JSON o tabelle.

Vincoli espliciti

Specifica formato, lunghezza, stile, cosa NON fare. Il modello risponde meglio a prompt diretti con vincoli ben definiti all'inizio.

Iterazione

Il primo prompt non è mai perfetto. Riformula, cambia ordine, aggiungi contesto finché l'output è quello atteso.

Anatomia del nostro prompt
Ruolo
Agisci come un recruiter con capacità di analisi precisa e strutturata di Curriculum Vitae.
Obiettivo
Estrai i dati: Nome, Età, Istruzione, Anzianità, Ruolo, Competenze, Lingue, Soft Skills, Tecnologie, Certificazioni.
Formato
Output esclusivamente JSON nello schema fornito. Nessuna frase introduttiva o conclusiva.
Vincoli
Non auto-completare. Null per numeri, "" per stringhe, [] per array vuoti.
Verifica
Controlla uno per uno tutti i campi: zero discrepanze tra tabella e JSON.

Sezione 05 — dettaglio

L'evoluzione iterativa dei prompt

Tre versioni, ognuna più precisa e completa della precedente.

Versione Output Caratteristiche chiave Limite principale
Prompt Tabellav1 Tabella Markdown Ruolo recruiter, 10 campi, "mancante" se assente. Nessun testo introduttivo. Solo tabella — non parsabile automaticamente da Python.
Prompt JSONv2 JSON strutturato Schema JSON fisso, null / "" / [] per dati mancanti, chiavi minuscole. Nessuna tabella leggibile — output solo machine-readable.
Prompt Unicov3 Tabella + JSON Combina v1 e v2. Prima tabella, poi JSON. Controllo incrociato obbligatorio. Più lungo e complesso — richiede context window ampia.
Estratto dal Prompt Unico (v3) — vincolo di verifica
# Istruzione di verifica alla fine del prompt "Terminato l'output torna sui tuoi passi e controlla  uno per uno i parametri della tabella in relazione  a quelli del file JSON: i parametri devono essere identici,  zero discrepanze. Se ci sono, torna a rileggere il CV"

Conclusioni

Cosa abbiamo imparato

Tre mesi di studio e progettazione — dall'AI theory al prodotto reale.

01

Il prompt è tutto

La qualità dell'output dipende direttamente dalla precisione del prompt.

02

Iterare è la norma

Il primo prompt non è mai perfetto. L'ottimizzazione progressiva è il cuore del prompt engineering.

03

La struttura salva

JSON schema fisso + verifica campo per campo = risultati affidabili e confrontabili automaticamente.

04

GitHub è essenziale

Versioning, branch e pull request hanno reso il lavoro a due persone ordinato, tracciabile e reversibile.

05

Verifica sempre l'AI

DeepDiff di Python ha dimostrato che i modelli sbagliano su dati strutturati.

06

L'AI potenzia, non sostituisce

L'AI accelera l'analisi dei CV, ma serve un prompt efficace, per generare risultati attendibili e prevedibili.
